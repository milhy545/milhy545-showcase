Direct MCP Bridge

A divergent branch that could not survive as a codebase, but still contributed one crucial mutation: explicit tool-to-endpoint routing instead of naive blind proxying.

What it was

The branch explored a simpler, more direct model: map a tool to a service and an endpoint intentionally, then call that route directly instead of pretending every backend supports a universal interface. It was blunt, but honest.

Why it changed

As a branch, it became a parallel timeline. It diverged too far, deleted too much modern infrastructure, and could not be merged safely. But the diagnosis it made was correct: generic forwarding was causing real routing failures.

What survived

The living descendant is the current adapter logic in the orchestrator. Today, tools are routed through service-specific mappings because this old branch proved that "one fake universal endpoint" was the wrong abstraction.

Curator Note

This is the ideal museum specimen: the branch died, the idea lived.

Known Descendants
